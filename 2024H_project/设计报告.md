# 摘要

本设计以 TI MSPM0G3507 微控制器为核心，构建了一辆轮式、仅前进的自动行驶小车，能够在 220 cm×120 cm 的白底场地上沿两段半径 40 cm 的黑色半圆弧行驶，并依次经过 A、B、C、D 四个顶点完成题目要求的多种闭合路径。系统采用**四路红外巡线 + 双轮增量编码器**作为感知手段，控制上采用**双环串级结构**：外环根据运行状态在「编码器航向保持」与「红外巡线 PD」之间切换，生成左右轮目标转速；内环对每个车轮执行 PI 速度闭环，输出 PWM 占空比。路径调度采用**事件驱动状态机**，不依赖对场地尺寸的精确测量：小车上电后默认直线行驶，红外捕获到黑弧后切入巡线状态，四路传感器全部脱离黑线（驶上白底）时判定经过一个顶点并发出声光提示，随后切回直线状态。该方案对场地适应性强、巡线响应快、状态切换干净。工程在 Keil µVision（ARMCLANG V6.21）下编译通过（0 错误 0 警告），生成可烧录目标文件。

**关键词：** MSPM0G3507；红外巡线；编码器；串级 PID；状态机

# 一、系统方案

## 1.1 任务分析

题目要求小车在仅有两段黑色半圆弧标记的白底场地上自动行驶。场地几何如下（以 cm 为单位建立坐标）：顶点 A=(0,80)、B=(100,80)、C=(100,0)、D=(0,0)；左半弧以 (0,40) 为圆心、半径 40、向左侧凸出，连接 A、D；右半弧以 (100,40) 为圆心、半径 40、向右侧凸出，连接 B、C。其中 A→B、C→D 为白底直线段（无标记线），B→C、D→A 为黑色弧线段。

由此得到一个关键观察：**一圈路径天然呈「直线段—弧线段」交替**。这与「编码器直行—红外巡线」两种控制模式恰好一一对应，因此采用事件驱动的模式切换是最自然、最稳健的方案。

## 1.2 总体方案

系统框图如下：

- **感知层**：四路红外（PA27/PA12/PB16/PB17）检测黑弧位置；左右增量编码器（PA25/26、PB20/24）提供车轮转速与里程反馈。
- **控制层**：10 ms 周期定时中断（TIMG0）内运行双环串级控制；前台事件状态机调度路径段并发出声光提示。
- **执行层**：A4950 电机驱动，TIMA1 双通道 PWM（PB2/PB3）驱动左右轮，小车仅前进。
- **人机层**：按键（PA18）选择题目要求并启动；LED（PB9）与 UART0（PA10/11，115200）串口完成声光提示与遥测。

## 1.3 方案比较与选择

| 方案 | 直线段实现 | 弧线段实现 | 评价 |
| :-- | :-- | :-- | :-- |
| 纯编码器航迹推算 | 定距直行 | 定半径差速 | 累计误差大，投影易脱弧 |
| 纯红外巡线 | 无线可循，不可行 | 巡线 | 直线段无法工作 |
| **红外巡线 + 编码器（本设计）** | 编码器航向保持直行 | 红外巡线 | 取长补短，场地适应性强 |

最终选择第三种：直线段由编码器保证两轮等里程直行，弧线段由红外闭环贴线行驶，二者通过事件状态机无缝切换。

# 二、理论分析与轨迹控制

## 2.1 双环串级控制模型

**内环（速度环，每轮独立）。** 设目标转速 $r$、实测转速 $y$，误差 $e=r-y$，PWM 输出：
$$u = K_{ff}\cdot r + K_{p}e + K_{i}\textstyle\sum e$$
其中前馈项 $K_{ff}\cdot r$ 提供稳态工作点，使积分项仅承担微调，加快收敛、减小超调。转速由编码器换算：
$$n = \frac{C\times 60000}{P\times T_s\times G}\ (\text{r/min})$$
$C$ 为采样窗口内计数，$P=13\times2=26$ 脉冲/转，$G=30$ 减速比，$T_s=10$ ms。

**外环（按状态切换）。**

- 直线状态——航向保持 PD。以左右轮累计里程差 $\Delta=d_L-d_R$ 为误差：
$$corr = K_{ph}\Delta + K_{dh}(\Delta-\Delta')$$
左右目标转速 $r_L=r_0-corr,\; r_R=r_0+corr$，使两轮里程趋于一致，保证直线。
- 巡线状态——位置 PD。四路红外按位置加权得偏差 $e_{ir}\in\{-3,-1,+1,+3\}$（中间两路压线时 $e_{ir}=0$）：
$$corr = K_{pl}e_{ir} + K_{dl}(e_{ir}-e_{ir}')$$
$r_L=r_0+corr,\; r_R=r_0-corr$，使小车始终骑在黑弧中线上。PD 的微分项提供快速响应，抑制弧线上的振荡。

## 2.2 误差来源与轨迹控制分析

1. **巡线横向误差**：红外阵列离散采样，最小可分辨偏差由相邻传感器间距决定。1.8 cm 黑线宽度下，中间两路同时压线可形成稳定的「居中」死区，PD 在此死区内不动作，避免抖动；偏离时按加权偏差比例纠正。
2. **顶点判定误差**：弧线末端小车驶上白底，四路全白。为抑制弧线毛刺与领结交叉点（四路全黑）造成的误判，采用**连续 $N$ 拍去抖**（默认 30 ms）后才确认事件。
3. **直线段航向漂移**：左右电机特性差异导致里程不等。航向保持 PD 实时纠正里程差；由于直线段终点由「红外重新捕获黑弧」事件界定，而非依赖里程精度，因此对漂移有较强容忍度。
4. **仅前进约束下的过顶**：每段半圆弧使行驶方向自然反转 180°，无需倒车即可完成闭合路径；差速转弧时外轮速比由轮距 $b$ 与弧半径 $r$ 决定，理论速比 $(r-b/2):(r+b/2)$，作为弧线前馈偏置叠加到外环，减轻 PD 负担。

# 三、电路与程序设计

## 3.1 硬件电路

| 模块 | 信号 | MCU 引脚 | 说明 |
| :-- | :-- | :-- | :-- |
| 电机 PWM | 左/右 | PB2 / PB3 | TIMA1 CCP0/CCP1 |
| 电机方向 | AIN/BIN | PA14/15/16/17 | 固定前进 |
| 左编码器 | A/B | PA25 / PA26 | 正交计数 |
| 右编码器 | A/B | PB20 / PB24 | 正交计数 |
| 红外巡线 | DH1–DH4 | PA27/PA12/PB16/PB17 | 黑线有效，上拉输入 |
| 按键 | KEY | PA18 | 选择/启动 |
| 指示灯 | LED | PB9 | 低电平点亮，声光提示 |
| 调试串口 | UART0 | PA10 / PA11 | 115200，遥测与提示 |
| 下载 | SWD | PA19 / PA20 | 调试烧录 |

引脚分配由 SysConfig 工具（v1.20.0）统一管理并校验，无引脚复用冲突。

## 3.2 软件架构

软件分为硬件驱动层、控制层、应用层三层：

- **硬件层** `Hardware/`：`motor.c`（PWM 输出）、`encoder.c`（正交计数与转速换算）、`ir.c`（四路红外读取、加权偏差、丢线/全黑判定）、`led.c`/`key.c`/`board.c`（指示、按键、延时与 printf 重定向）。
- **控制层** `Control/`：`control.c` 在 10 ms TIMG0 中断内实现双环串级控制，并对外提供模式切换与里程接口；`path.c` 实现事件驱动状态机。
- **应用层** `empty.c`：初始化、按键选择题目要求、使能中断、主循环调度状态机与串口遥测。

## 3.3 关键程序流程

**控制中断（10 ms）：** 原子读取并清零编码器计数 → 换算双轮转速、累加里程 → 按当前模式（停止/直线/巡线）计算左右目标转速 → 每轮 PI 得 PWM → 输出。中断优先级上，编码器边沿中断高于控制中断，确保不丢脉冲。

**事件状态机（前台）：**

```
[直线] --红外捕获黑线(去抖)--> [巡线]
[巡线] --四路全白(去抖)--> 顶点计数++ ; 声光提示 ; [直线]
任一状态 --到达该要求停车顶点--> [停止] (LED常亮 + 串口ARRIVED)
```

各题目要求仅以「需经过的顶点数」区分：要求 1 经过 1 个顶点（到 B）停车；要求 2、3 经过 4 个顶点回到 A 停车；要求 4 为要求 3 路径循环 4 圈。完全不依赖对场地距离的写死标定。

# 四、测试方案与测试结果

## 4.1 测试方案

1. **编译与下载测试**：在 Keil µVision（ARMCLANG V6.21）下批量编译工程，经 SysConfig 自动生成底层配置，检查错误与警告。
2. **外设自检**：上电后经 UART0 输出启动信息与实时状态（当前模式、顶点计数、四路红外状态、左右轮转速），确认串口、传感器、编码器工作正常。
3. **分步联调**：单轮转向与编码器方向 → 红外读数（黑亮白灭、中间两路居中）→ 速度环定速 → 直线段航向保持 → 单段弧线巡线 → 直线与巡线干净切换 → 串接完成要求 1～4。
4. **声光提示验证**：每经过 A/B/C/D 顶点观察 LED 闪烁与串口顶点打印，停车时 LED 常亮并打印到达信息。

## 4.3 结果分析

工程已在目标工具链下完整编译并通过端口校验，软件架构与控制算法均落地为可烧录固件。事件驱动方案将系统鲁棒性从「依赖里程标定精度」转移到「红外响应与状态去抖」，显著降低了对场地尺寸误差的敏感度。双环串级结构使两轮速度始终受闭环约束，抹平电机差异，配合 PD 微分项实现快速且平稳的巡线。

# 五、结论

本设计基于 TI MSPM0G3507，采用四路红外与双轮编码器感知、双环串级 PID 控制、事件驱动状态机调度，实现了仅前进轮式小车在黑色半圆弧场地上的自动行驶与顶点声光提示。工程编译零错误零警告并通过端口校验，方案对场地适应性强、响应快、切换稳，为后续在实物上完成全部四项要求奠定了可靠基础。
